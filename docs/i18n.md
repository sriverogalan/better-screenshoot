# Internacionalización (i18n)

Better Screenshoot usa [vue-i18n](https://vue-i18n.intlify.dev/) en el frontend y ficheros JSON espejo para el menú nativo de la bandeja del sistema.

## Idiomas soportados

| Código | Estado |
|--------|--------|
| `en` | Completo (idioma fuente) |
| `es` | Completo |
| `fr` | Completo |
| `de` | Completo |
| `pt` | Completo |
| `it` | Completo |

## Añadir o completar un idioma

1. Copia [`apps/desktop/src/locales/en.json`](../apps/desktop/src/locales/en.json) a `apps/desktop/src/locales/xx.json`.
2. Traduce **solo los valores**; no cambies las claves.
3. Añade `xx` a `AppLocale` en [`packages/shared-types/src/index.ts`](../packages/shared-types/src/index.ts).
4. Registra el locale en [`apps/desktop/src/i18n/index.ts`](../apps/desktop/src/i18n/index.ts) (`SUPPORTED_LOCALES` y `messages`).
5. Añade la etiqueta en `settings.languageOptions` dentro de todos los ficheros de locale.
6. Crea `apps/desktop/src-tauri/locales/xx.json` con el namespace `tray` (copia de `en.json` en esa carpeta).
7. Actualiza el selector de idioma en `SettingsView.vue` si hace falta una entrada nueva.
8. Ejecuta la validación:

```bash
pnpm --filter @better-screenshoot/desktop i18n:check
```

## Estructura de claves

Los textos están organizados por namespace:

- `common`, `nav`, `history`, `settings`, `editor`, `overlay`, `captureWindow`
- `licensing` — nombres y features de planes (los precios siguen en `@better-screenshoot/licensing`)
- `errors` — mensajes del backend (códigos emitidos por Tauri)
- `tray` — menú de bandeja (duplicado en `src-tauri/locales/`)

## Mensajes del backend

Rust emite códigos (`AppErrorPayload`) en lugar de frases en inglés. El frontend los traduce con `translateAppError()` y `translateMessageCode()`.

Al añadir un error visible al usuario:

1. Añade el código en Rust (`errors.rs` o el módulo correspondiente).
2. Añade la clave en `errors.*` de **todos** los ficheros en `src/locales/`.

## CI

El workflow de CI ejecuta `pnpm --filter @better-screenshoot/desktop i18n:check` junto al typecheck del frontend.
